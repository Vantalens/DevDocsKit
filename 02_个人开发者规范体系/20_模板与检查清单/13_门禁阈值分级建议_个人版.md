# 门禁阈值分级建议（个人版）

版本：v1.0.1
状态：生效
最后更新：2026-06-27

## 1. 目标
按个人能力成长分阶段提高质量阈值，保证可执行与可持续。

## 2. 分级
- S1 起步期
- S2 稳定期
- S3 强化期

## 3. 指标建议
| 指标 | S1 | S2 | S3 |
|---|---|---|---|
| 单元覆盖率 | >= 40% | >= 55% | >= 70% |
| 核心流程通过率 | 100% | 100% | 100% |
| P1 通过率 | >= 90% | >= 95% | >= 98% |
| 接口 P95 | <= 1000ms | <= 800ms | <= 600ms |
| 发布前检查 | 100% | 100% | 100% |

说明：本表的单元覆盖率是个人项目成熟度爬坡指标，不替代 `10_核心规范/08_测试验收与发布规范_个人版.md` 中的检查覆盖率 >= 80% 发布门禁。低成熟度项目未达到 80% 检查覆盖率时，只能以阻断、豁免或补救计划收尾，不得宣称完整发布达标。

## 4. 升级条件
- 连续 2 次迭代达标可升级一级

## 5. 降级条件
- 连续 2 次发布出现阻断或回滚失败

## 6. 变更记录
- v1.0.1：明确成长型单元覆盖率阈值不降低 80% 检查覆盖率发布门禁。
- v1.0.0：首版。
