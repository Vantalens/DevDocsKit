# AI 协作与会话记忆规范（个人版）

版本：v1.3.0
状态：生效
最后更新：2026-06-27

## 1. 目标
保证多次会话中的上下文连续与规则一致，降低重复错误。

## 2. 三层记忆文件

个人项目采用三层记忆：

- progress：当前状态、下一步、阻塞项和正在处理的任务。
- lessons：可复用错误模式、纠正规则和防复发检查点。
- 阶段归档：阶段目标、验证证据、代码审核结论、issue/PR 链接、发布或回滚记录。

核心文件职责：
- AGENT_RULES：AI 行为规则与约束
- progress：当前进度与下一步
- lessons：错误模式与防复发规则
- 阶段归档：阶段证据链、审核结论、issue/PR 链接和收尾报告

## 3. 会话启动协议

AI 默认作为执行代理：按已冻结文档和实施计划连续推进普通任务，不反复打断用户。遇到安全、数据迁移、删除、依赖升级、架构边界、生产发布、关键代码提交等高风险动作，必须停下确认。

每次会话必须先：
1. 读取 AGENT_RULES
2. 读取 progress
3. 读取相关 PRD/APP_FLOW/IMPLEMENTATION_PLAN
4. 检查当前任务是否匹配本地 Codex 已支持 skill、插件或 MCP 工具
5. 读取项目开发文档：TECH_STACK、TEST_PLAN、AGENT_RULES 或等价文件
6. 使用 codegraph 或等价本地索引梳理目标模块和影响面
7. 输出本次任务边界、主模块、项目类型、可用 skill、验收条件、权限边界和代码审核计划

## 4. 会话执行协议
- 一次只做一个任务
- 不越界修改无关模块
- 发现规则冲突先修正文档
- 优先使用本地已支持 skill；GitHub star 中的外部 skill 只作为候选方法论，采纳前必须记录来源、风险、验证方式和退出条件
- 代码理解、研究检索、代码审核、数据分析、部署等任务必须先选择匹配能力，再执行文件修改或命令
- 对会写文件、访问网络或读取敏感信息的 skill，必须先说明最小权限范围
- AI 不得过度超前，不得实现当前计划之外的复杂功能
- 开发必须按模块化方式推进，跨模块修改必须记录影响范围
- 代码审核发现的 P0/P1、安全或阻断当前用户路径的问题在当前阶段修复；非阻断问题进入 issue 和本地阶段报告，后续单独处理

## 5. 会话收尾协议
- 更新 progress
- 更新 lessons
- 更新阶段归档：验证证据、审核结论、issue/PR 链接、未覆盖风险
- 给出下一会话建议输入
- 若本轮启用或停用 skill，记录原因、效果、失败点和后续是否继续使用
- 阶段完成后执行代码审核，并把问题按标签提交 issue 或给出可提交 issue 内容，同时写入本地阶段报告
- 报告检查覆盖率是否大于等于 80%
- 记录 codegraph 梳理结论和遗留风险

## 6. 评审清单
- 是否先读后做
- 是否完成收尾更新
- 是否沉淀新经验
- 是否优先检查 Codex 已支持 skill
- 是否把外部 skill 的候选价值与项目内已验证事实分开记录
- 是否阅读项目开发文档并使用 codegraph 梳理影响面
- 是否完成阶段代码审核和 issue 标签化报告
- 是否达到 80% 检查覆盖率
- 是否避免 AI 过度超前
