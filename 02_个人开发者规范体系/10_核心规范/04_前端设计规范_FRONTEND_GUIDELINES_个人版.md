# 前端设计规范 FRONTEND_GUIDELINES（个人版）

版本：v1.1.0  
状态：生效  
最后更新：2026-06-09

## 1. 目标
以最小规则建立视觉一致性、交互可用性和移动端兼容性。

## 2. Design Tokens
必须定义：
- 颜色：主色、语义色、中性色
- 字体：标题、正文、注释
- 间距：4 的倍数刻度
- 圆角与阴影
- 动效时长

## 3. 响应式规则
- Mobile First
- 固定断点并文档化
- 核心组件需给出移动端与桌面端行为

## 4. 组件规范
每个核心组件记录：
- 责任边界
- 输入输出
- 状态（默认/加载/错误/禁用）
- 可访问性要求

## 5. UX 最小标准
- 空态可引导
- 错误态可恢复
- 加载态可感知
- 关键操作有反馈
- 默认采用现代、清晰、可访问、响应式 UI 风格，除非用户明确要求其他风格
- 页面内容符合用户使用习惯，主任务路径优先
- 禁止复杂功能堆叠、非人性化设计和无必要多步操作
- AI 不得提前添加当前计划之外的复杂交互或未来功能

## 6. 评审清单
- 是否使用统一 token
- 是否覆盖关键状态
- 是否完成移动端验证
- 是否符合现代 UI 和用户使用习惯
- 是否避免复杂功能和非人性化设计

## 7. 变更记录
- v1.1.0：新增现代 UI 默认风格、用户习惯、禁止复杂化和 AI 开发节制要求。
- v1.0.0：首版。
