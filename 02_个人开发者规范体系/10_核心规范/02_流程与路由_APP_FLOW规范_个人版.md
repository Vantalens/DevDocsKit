# 流程与路由 APP_FLOW 规范（个人版）

版本：v1.0.0  
状态：生效  
最后更新：2026-04-19

## 1. 目标
定义用户路径与状态流转，避免页面跳转和交互逻辑失控。

## 2. 页面与路由表
每个页面必须记录：
- 页面名
- 路由
- 入口
- 退出路径
- 权限

## 3. 流程定义
每条流程必须包含：
- FLOW ID（FLOW-001）
- 触发条件
- 成功路径步骤
- 失败路径步骤
- 终态

## 4. 异常流清单
最低必须覆盖：
- 未登录
- 参数非法
- 网络错误
- 资源不存在

## 5. 状态机建议
使用 状态 -> 事件 -> 新状态 的描述方式，减少口头约定。

## 6. 评审清单
- 是否有成功流和失败流
- 是否有死循环或不可达页面
- 是否定义了超时后行为
