## 3. 阅读顺序

### 3.0 项目类型分流

个人项目开始前先选择项目类型，并追加对应检查项：

- Web 全栈：确认前端、后端、数据库、鉴权、部署、端到端用户路径和回滚。
- 桌面/本地工具：确认 GUI 入口、本地运行时、文件读写、打包安装、离线复现和用户数据边界。
- AI/Agent 工具：确认模型调用、工具权限、任务编排、记忆、成本、可审计输出和外部服务边界。

个人路径默认流程：关键闭环冻结 -> 小阶段实施 -> 自动化测试和真实路径验证 -> 阶段归档 -> 确认后提交。

关键闭环包括：目标、范围、流程、技术栈、实施阶段、测试验收、回滚和 AI 执行边界。完整规格允许在后续阶段补充，但每个阶段开始前必须重新冻结本阶段范围和验收标准。

## 3. 阅读顺序

最小阅读路径：

1. `10_核心规范/01_需求文档_PRD规范_个人版.md`
2. `10_核心规范/06_实施计划与复现_IMPLEMENTATION_PLAN_个人版.md`
3. `10_核心规范/07_AI协作与会话记忆规范_AGENT_PROGRESS_LESSONS_个人版.md`
4. `10_核心规范/08_测试验收与发布规范_个人版.md`

涉及前端、后端、架构或 GUI 项目时，再补读对应核心规范和模板。

## 4. 三道门禁

个人项目允许按小阶段推进，但每个小阶段必须能对应一段可验收用户路径、一个可独立审查的文档阶段，或一个可验证修复集合。

## 4. 三道门禁

- Gate-1 进入开发：PRD、APP_FLOW、IMPLEMENTATION_PLAN 完整且可测。
- Gate-2 进入发布：核心流程通过，回滚方案演练完成。
- Gate-3 进入稳定：上线后监控正常，复盘结论闭环。

## 5. Definition of Ready

- 本轮目标与范围明确。
- 任务拆分完成且可独立验收。
- 环境与依赖版本锁定。
- AI 协作规则、进度记录和验证方式明确。

## 6. Definition of Done

- 代码、测试、文档一致。
- progress、lessons 与阶段归档已更新。
- 可在干净环境复现关键结果。
- 阶段代码审核、issue 记录和覆盖率检查已完成。
