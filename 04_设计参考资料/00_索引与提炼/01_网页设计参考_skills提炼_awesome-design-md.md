# 网页设计参考 Skills 提炼（基于 awesome-design-md）

版本：v1.0.0  
状态：生效

## 1. 文档目的
基于 VoltAgent 的 awesome-design-md 仓库，提炼可复用、可落地的网页设计 skills，作为个人或团队进行 AI 协作设计与实现时的参考。

参考来源：
- https://github.com/VoltAgent/awesome-design-md
- https://raw.githubusercontent.com/VoltAgent/awesome-design-md/main/README.md

## 2. 核心认知
该仓库的关键方法不是抄具体页面，而是使用 DESIGN.md 作为设计约束层，让 AI 在以下九个维度稳定输出：
1. Visual Theme 与 Atmosphere
2. Color Palette 与 Roles
3. Typography Rules
4. Component Stylings
5. Layout Principles
6. Depth 与 Elevation
7. Do 与 Dont
8. Responsive Behavior
9. Agent Prompt Guide

结论：
- 真正高价值 skill 是先定义规则，再让 AI 生成界面。
- 先有设计约束，后有页面实现，能显著降低风格漂移。

## 3. 可直接使用的 24 个高价值 Skills

### A. 风格定位 Skills
1. 先定情绪，再定组件。先写页面要传达的感觉，如专业、速度感、可信赖。
2. 每个项目只选 1 到 2 个主视觉语言，避免混搭失控。
3. 为品牌建立一句话视觉原则，例如 黑白高对比 + 精准排版 + 轻动效。
4. 通过行业风格映射加速选型：
- 开发者工具：深色、信息密度高、代码感
- 金融支付：高对比、强秩序、可信赖
- 创意产品：更强色彩、更多动效、形态更自由

### B. 色彩系统 Skills
5. 颜色按角色定义，而不是按感觉命名。使用 主色、强调色、成功、警告、错误、文本、边框、背景、表面。
6. 每个角色至少定义亮色和暗色两套值。
7. 控制强调色数量，通常 1 主 1 辅即可。
8. 高信息密度页面优先使用中性色层级区分，而不是依赖大量彩色区块。

### C. 排版与信息层级 Skills
9. 先定义字重梯度，再定义字号梯度，确保层级稳定。
10. 标题和正文分工明确：标题用于结构，正文用于阅读效率。
11. 长文档页面优先阅读优化风格，使用更大行高和更稳定的段落间距。
12. 数据型页面优先紧凑排版，保留必要留白避免疲劳。

### D. 组件规范 Skills
13. 每个核心组件必须定义完整状态：默认、悬停、激活、禁用、加载、错误。
14. 按组件角色统一边角、阴影、描边，避免每个模块自定义。
15. CTA 组件只保留一种主样式，其他动作使用次样式。
16. 导航组件优先可预测性，避免炫技交互影响定位。

### E. 布局与密度 Skills
17. 先定义布局骨架，再填内容。建议先定容器、网格、间距刻度。
18. 对首页使用叙事型布局，对控制台使用任务型布局。
19. 通过区块节奏控制阅读速度：重信息区紧凑，转场区留白。
20. 大图风格站点必须搭配强标题层级，不然信息会被图像吞没。

### F. 深度与动效 Skills
21. 阴影和模糊仅用于层级表达，不用于装饰堆叠。
22. 动效优先表达因果关系，如加载、切换、反馈，而非纯展示。
23. 首屏动效应控制时长，避免影响可读和性能。

### G. 响应式与可访问性 Skills
24. Mobile First 先定义最小可用界面，再扩展桌面版本。关键操作必须保证触控目标大小和可读性。

## 4. 风格参考库（按场景提炼）

### 4.1 开发者工具与 AI 平台
适合：后台、控制台、文档站、开发者产品官网。
可借鉴方向：Vercel、Cursor、Supabase、Sentry、Linear、Mintlify。
共性：深浅层级清晰、信息结构严谨、排版偏理性、强调效率。

### 4.2 创意与营销页面
适合：品牌官网、活动页、产品发布页。
可借鉴方向：Framer、Webflow、Figma、Clay、Nike、Airbnb。
共性：视觉冲击力高、动效与摄影更强、情绪化表达明显。

### 4.3 金融与交易类页面
适合：支付、账单、资产、交易相关产品。
可借鉴方向：Stripe、Mastercard、Coinbase、Wise、Revolut。
共性：可信赖优先、信息秩序强、风险提示明确、对比清晰。

### 4.4 内容与媒体页面
适合：博客、知识库、资讯站、文档中心。
可借鉴方向：Notion、WIRED、The Verge、Pinterest。
共性：阅读体验优先、视觉节奏明确、内容结构主导界面。

## 5. AI 设计提示词 Skills 模板

### 5.1 页面级模板
请按以下约束生成页面：
- 视觉气质：
- 色彩角色：主色、强调色、背景、文本、边框
- 排版层级：H1 到 Body 的字号与字重关系
- 组件状态：按钮、输入框、卡片、导航的全状态
- 布局规则：容器宽度、网格列数、间距刻度
- 响应规则：移动端优先，断点行为明确
- 禁止项：避免颜色漂移、避免多套按钮风格

### 5.2 组件级模板
请实现一个可复用组件，并满足：
- 角色：主操作或次操作
- 状态：默认、悬停、激活、禁用、加载、错误
- 可访问性：键盘可操作，语义标签完整
- 视觉一致性：遵循既有 token，不新增私有样式

### 5.3 调优级模板
请对现有页面做一致性修复：
- 对齐色彩角色
- 对齐排版梯度
- 对齐组件状态
- 提升移动端可用性
- 输出修改清单与理由

## 6. 快速落地流程
1. 先确定场景与风格参考库
2. 写一页 DESIGN 约束文档（九维结构）
3. 先做 1 页和 4 个核心组件
4. 用统一提示词做扩展生成
5. 上线前做一致性审查与移动端回归

## 7. 常见误区
- 误区 1：只说现代感，不给规则
- 误区 2：只看单页视觉，不管组件状态
- 误区 3：先生成全站，再补设计系统
- 误区 4：暗色页面只追求酷，不考虑可读性
- 误区 5：动效过量，影响信息传达

## 8. 你的项目建议起手式
- 若是开发者产品：优先参考 Vercel + Linear + Supabase 路线
- 若是营销官网：优先参考 Framer + Webflow + Nike 路线
- 若是内容平台：优先参考 Notion + WIRED + Mintlify 路线
