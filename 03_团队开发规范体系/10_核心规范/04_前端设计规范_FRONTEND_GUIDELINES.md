# 前端设计规范 FRONTEND_GUIDELINES

版本：v1.1.0  
状态：生效  
最后更新：2026-06-09

## 1. 目标
建立一致、可扩展、可访问、可复现的前端体验规范。

## 2. 设计系统基础
### 2.1 Design Tokens
必须定义：
- 色板：主色、辅助色、语义色、中性色
- 字体：标题、正文、说明
- 间距：4 的倍数刻度
- 圆角、阴影、层级
- 动画时长与缓动曲线

### 2.2 断点与布局
- Mobile First
- 断点必须固定并文档化
- 网格规则与容器宽度必须统一

### 2.3 组件规范
每类组件需定义：
- 用途与边界
- 状态集合（默认、悬停、激活、禁用、加载、错误）
- 交互反馈与无障碍要求
- 最小可用尺寸

## 3. UX 规则
- 关键动作必须显著
- 失败态必须可恢复
- 空状态必须有引导
- 加载态必须可感知
- 默认采用现代、清晰、克制、可访问、响应式的 UI 风格，除非需求明确指定其他风格
- 页面内容必须符合目标用户使用习惯，优先支持主任务完成
- 禁止复杂功能堆叠、非人性化流程、隐藏关键操作和无必要的多步确认
- AI 不得为了展示能力而添加当前需求之外的复杂交互或未来功能

## 4. 可访问性
最低要求：
- 颜色对比度满足 WCAG 基线
- 可键盘操作
- 表单控件具备语义标签
- 动效可被降低

## 5. 性能规则
- 首屏资源预算
- 图片与字体优化策略
- 关键渲染路径约束

## 6. 设计到实现映射
- 每个 UI 模块必须映射到组件目录结构
- 禁止一次性页面样式堆叠，不可复用样式必须说明原因

## 8. 评审清单
- 是否使用统一设计令牌
- 是否覆盖关键交互状态
- 是否通过移动端与桌面端验收
- 是否符合现代 UI 风格和用户使用习惯
- 是否避免复杂功能和非人性化设计
- 是否覆盖 UX、性能和可访问性审核 issue
