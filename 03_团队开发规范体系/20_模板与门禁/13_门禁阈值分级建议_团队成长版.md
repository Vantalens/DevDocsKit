# 门禁阈值分级建议（团队成长版）

版本：v1.0.0  
状态：生效  
最后更新：2026-04-19

## 1. 目标
根据团队成熟度分阶段提升质量门禁，保证可执行性与持续改进。

## 2. 分级模型
- G1：起步期（可交付优先）
- G2：稳定期（质量与速度平衡）
- G3：强化期（质量优先）

## 3. 指标阈值建议
| 指标 | G1 | G2 | G3 |
|---|---|---|---|
| 单元测试覆盖率 | >= 50% | >= 65% | >= 75% |
| 核心模块覆盖率 | >= 70% | >= 80% | >= 90% |
| P0 场景通过率 | 100% | 100% | 100% |
| P1 场景通过率 | >= 90% | >= 95% | >= 98% |
| 接口 P95 | <= 800ms | <= 600ms | <= 500ms |
| 发布前检查完成率 | 100% | 100% | 100% |
| 回滚演练频率 | 每双周 | 每周 | 每周 |

## 4. 升级条件
从 G1 升级 G2：
- 连续 2 个迭代无 P0 线上事故
- 追踪矩阵覆盖率连续达标

从 G2 升级 G3：
- 连续 3 个迭代质量门禁达标
- MTTR 指标稳定下降

## 5. 降级条件
满足任一条件可降级：
- 连续 2 个迭代出现发布阻断
- 连续 2 次回滚失败
- 文档与实现偏差无法在迭代内收敛

## 6. 执行建议
- 每个季度审查一次分级
- 阈值上调不超过 10%，避免组织震荡
- 若业务高峰期可临时冻结阈值
