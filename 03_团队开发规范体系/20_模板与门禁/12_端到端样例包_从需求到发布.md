# 端到端样例包：从需求到发布

版本：v1.0.0  
状态：样例  
最后更新：2026-04-19

## 1. 目标
提供一套可复制的最小闭环样例，帮助团队快速理解并执行规范体系。

## 2. 样例背景
场景：用户登录后查看个人资料页，并支持更新昵称。

## 3. 需求样例（PRD）
- REQ-001：用户可使用邮箱密码登录系统
- REQ-002：登录后可查看个人资料
- REQ-003：用户可更新昵称并立即生效

验收标准（示例）：
- Given 用户输入正确账号密码，When 点击登录，Then 3 秒内进入资料页
- Given 用户在资料页修改昵称，When 点击保存，Then 页面提示成功且昵称刷新

## 4. 流程样例（APP_FLOW）
- FLOW-001：登录成功流
- FLOW-002：登录失败流（账号密码错误）
- FLOW-003：资料更新成功流
- FLOW-004：资料更新失败流（网络超时/昵称非法）

## 5. 实施任务样例（IMPLEMENTATION_PLAN）
- TASK-001：实现登录页面表单与校验（对应 REQ-001）
- TASK-002：实现登录 API 与鉴权中间件（对应 REQ-001）
- TASK-003：实现资料页读取接口与渲染（对应 REQ-002）
- TASK-004：实现昵称更新接口（对应 REQ-003）
- TASK-005：实现资料页更新交互与反馈（对应 REQ-003）

## 6. 接口样例（BACKEND_STRUCTURE）
- API-001：POST /api/auth/login
- API-002：GET /api/profile
- API-003：PATCH /api/profile/nickname

错误码示例：
- AUTH_401：认证失败
- PROFILE_400：参数非法
- PROFILE_409：并发更新冲突

## 7. 测试样例（TEST）
- TEST-001：登录成功
- TEST-002：登录失败提示
- TEST-003：资料页加载成功
- TEST-004：昵称更新成功
- TEST-005：昵称非法校验
- TEST-006：网络超时重试提示

## 8. 发布样例（REL）
