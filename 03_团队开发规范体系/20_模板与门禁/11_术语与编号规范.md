# 术语与编号规范

版本：v1.0.0  
状态：生效  
最后更新：2026-04-19

## 1. 目标
统一跨文档术语与编号体系，降低沟通歧义，提升追踪与复现效率。

## 2. 术语统一
- 需求：REQ
- 流程：FLOW
- 任务：TASK
- 接口：API
- 测试：TEST
- 发布：REL
- 架构决策：ADR

规则：
- 同一概念只能有一个主术语
- 首次出现可附英文缩写，后续统一用缩写

## 3. 编号规则
### 3.1 通用格式
- REQ-001
- FLOW-001
- TASK-001
- API-001
- TEST-001
- REL-001
- ADR-001

### 3.2 编号约束
- 编号全局唯一，不允许复用
- 禁止删除历史编号，可标记 deprecated
- 新增编号按序递增

## 4. 引用规则
- 任何 TASK 必须引用至少一个 REQ
- 任何 TEST 必须引用至少一个 REQ 或 FLOW
- 任何 REL 必须列出变更涉及的 TASK 与 TEST

## 5. 文档命名规则
- 采用 前缀序号_主题_类型.md
- 前缀序号使用两位数字
- 主题使用中文，类型使用固定英文或约定缩写

## 6. 字段命名一致性
跨文档字段统一使用：
- id
- owner
- reviewer
- status
- priority
- risk_level
- updated_at

## 7. 评审清单
- 是否存在重复编号
- 是否存在无来源术语
- 是否存在未映射编号链路
