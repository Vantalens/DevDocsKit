## 1. 定位

本文件是 Claude Code 在 DevDocsKit 中执行任务时的工具入口。为避免 `CLAUDE.md` 与 `AGENTS.md` 规则重复漂移，通用 AI 代理执行规则以根目录 `AGENTS.md` 为唯一主来源。

Claude Code 或兼容代理开始任务前必须先阅读：

1. `README.md`
2. `AGENTS.md`
3. `01_通用开发规范/02_文档维护SOP.md`
4. `01_通用开发规范/01_开发标准文档编写规则.md`
5. 与任务场景匹配的个人版或团队版入口和核心规范

长期维护、多模块或高风险任务还必须阅读：

- `01_通用开发规范/03_大型项目开发规范.md`
- 相关 `TECH_STACK`、`IMPLEMENTATION_PLAN`、`TEST_PLAN`、`progress`、`lessons` 或等价项目文档

## 2. 执行规则

- 执行顺序、路径分流、项目类型分流、模块化约束、codegraph 使用、阶段审核、issue+本地报告、覆盖率门禁、提交规则、版本规则和收尾要求均以 `AGENTS.md` 为准。
- 如 `CLAUDE.md` 与 `AGENTS.md` 发生冲突，以 `AGENTS.md` 为准，并将冲突登记为文档维护问题。
- 修改 AI 执行约束时，优先修改 `AGENTS.md`；只有 Claude Code 专属差异才写入本文件。

## 3. Claude Code 专属注意事项

- 优先使用当前环境已提供的本地工具、skill 或 MCP 能力；引入外部能力前按通用规范记录来源、权限、验证方式和停用条件。
- 不把 Claude Code 的临时会话输出当作项目事实来源；可复用结论必须沉淀到项目文档、issue、PR 描述或等价跟踪项。
- 若任务无法执行 `AGENTS.md` 要求的完整门禁，必须说明具体原因、替代检查方式和未覆盖风险。
